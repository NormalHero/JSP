
 1. 로컬 환경과 웹 호스팅 환경의 차이
  1-1. 웹 호스팅이란?
    - 웹 호스팅 -> 서버 환경을 구성하고, 그 환경을 임대하는 서비스
    - 개인이 서버 환경을 구성하기에는 매우 많은 비용과 노력이 필요하다.
    - 웹 호스팅을 이용하면 매달 이용료를 지불하고 서버 환경을 이용할 수 있다.
    - Whois, 나야나, 카페24

  1-2. 로컬 환경이란?
    - 작업을 목적으로 개발자 자신의 PC에 서버를 직접 설치한 상태

  1-3. 작업 환경 차이
    - 로컬 환경
	-> 개발자 자신의 컴퓨터에서 작업하기 때문에 초기 비용이 발생하지 않는다.
  	-> 외부와의 통신이 발생하지 않기 때문에 웹 호스팅 환경보다
	동작 속도가 빠르다.
	-> 작업 완료 후, 서비스를 위해서는 웹 호스팅 환경으로 결과물을 
	이전시켜야 한다.(배포)

   - 웹 호스팅
	-> 초기에 비용이 발생한다.
	-> 모든 작업이 원격에서 이루어진다.
	-> 작업 완료 후, 추가적인 이전 작업 없이 바로 서비스가 가능하다.

   1-4. HTML
     - HyperText Markup Language의 줄임말
     - 웹 문서를 만들기 위한 기본적인 웹 언어

   1-5. Java코드가 작성되는 스크립트 블록 <% %> 안에서는 Java가 지원하는 모든
   형태의 변수(기본 자료형 변수, 배열, 객체 등)를 선언 및 할당할 수 있다.

   1-6. 메서드 정의를 위한 <%! %> 블록의 사용
     - <%! %> 블록 안에서 정의한 메서드의 리턴값을 변수에 저장하고
     그 변수값을 HTML에 출력하거나, 메서드 자체를 <%= 메서드이름() %> 형태로
     HTML에 출력한다.

   1-7. 모듈화 된 웹 페이지 조립하기
 	<%@ include file="JSP 파일경로" %>

    - 모든 웹 페이지가 공통적으로 공유하는 영역을 별도의 파일 형태로 모듈화 하고,
    참조해서 사용할 수 있다.
    - include 페이지 지시자는 주어진 경로의 소스코드를 복사,붙여넣기 하는 형태로 
    동작하기 때문에 모든 페이지에서 공통적으로 나타나는 UI에 대한 코드를
    재사용하기 위해 사용한다.

  1-8. include 경로 지정 방식
    - 상대 경로 -> 현재 JSP 파일의 위치를 기준으로 하는 상대적 경로

    - 절대 경로 -> 경로명이 "/" 부터 시작될 경우, 프로젝트의 "WebContent" 디렉토리를
    		최상위 위치로 인식한다.

  1-9. CSS 적용시 id나 class 속성에 따른 스타일 적용
    - id 적용
	#아이디{ 속성1:속성값; 속성2:속성값; }

    - id 사용
	id="아이디"

    - class 적용
	.클래스명{ 속성1:속성값, 속성2:속성값; }

    - class 사용
	class="클래스명"

2. JSP 내장 객체
  2-1. JSP 내장 객체
    - JSP 페이지 내에서 제공되는 특수한 클래스 형의 객체
    - 내장 객체는 선언과 할당 없이 사용할 수 있다.
	-> JSP 페이지가 Java 클래스 형태로 변환될 때, JSP 컨테이너(톰캣)가
  	Java 클래스 안에 자동적으로 포함 시킨다.

  2-2. 변소의 유효성 범위
    - 변수는 자신이 선언된 {...} 범위 안에서만 유효하다.
    - JSP페이지들은 각각 Main 클래스에 대응 된다고 볼수 있다.
    - 때문에 서로 다른 JSP 페이지간에는 변수가 공유되지 않는다.
    - from.jsp 에서 to.jsp로 링크의 이동이 발생하면, to.jsp에서는 from.jsp에 선언된 
    변수를 인식할수 없다.

  2-3. GET 방식의 데이터 전송
    - URL에 변수의 이름과 값을 포함시키는 HTTP GET 방식의 데이터 전송
    - 페이지가 이동되는 과정에서 변수의 값이나 사용자의 입력값을
    다른 페이지에서도 인식할 수 있도록 하는 웹 프로그래밍 처리 기법
    - 사용방법
 	-> 페이지의 URL뒤에 "?"를 명시하고 "변수이름=값"의 형태로 데이터를
	추가한다.
	-> 두 개 이상의 값을 포함시켜야 할 경우 "&"로 구분할 수 있다.

	페이지URL?변수명=값&변수명=값&....&변수명=값

	http://localhost:18080/MyProject/hello.jsp?user_id=jspuser&age=20
